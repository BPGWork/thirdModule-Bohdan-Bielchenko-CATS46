Цей проєкт — вебзастосунок на Java Servlet/JSP, у якому користувач може:
- зареєструватися,
- увійти в акаунт,
- обрати історію,
- пройти її через систему виборів,
- отримати фінальний результат,
- переглянути свою статистику в профілі.

По суті це інтерактивна система текстових історій із розгалуженим сюжетом. Користувач рухається по фазах історії, 
обираючи один із доступних варіантів, а система переводить його до наступного етапу або до кінцівки.

Проект побудований навколо кількох сутностей. Це:
- User ( логін, пароль, кількість пройдених кінцівок, список отриманих кінцівок )
- History ( назва, інформація про історію )
- HistoryPhase ( заголовок поточного етапу, набір виборів користувача, переходи до наступних фаз )

Як працює логіка застосунку
1. Реєстрація ( RegistrationServlet ):
- приймає логін і пароль,
- перевіряє, чи існує користувач,
- хешує пароль,
- створює нового користувача,
- записує його в JSON,
- зберігає користувача в сесію та cookie.

2. Авторизація ( LoginServlet ):
- шукає користувача за логіном,
- перевіряє пароль,
- при успіху зберігає користувача в сесію і cookie,
- при помилці повертає на головну з параметром помилки.

3. Головна сторінка ( IndexServlet )
- намагається відновити користувача із cookie,
- завантажує список історій,
- дозволяє перейти в профіль,
- дозволяє обрати історію для проходження.

4. Сторінка історії ( HistoryServlet ):
- перевіряє, чи є в сесії вибрана історія,
- показує короткий опис,
- після натискання Start готує сесію до проходження.

5. Проходження історії ( PlayServlet ):
- бере історію із сесії,
- завантажує всі її фази,
- визначає поточну фазу,
- показує доступні варіанти вибору,
- після вибору переводить користувача до наступної фази,
- якщо досягнута кінцівка — зберігає результат користувача.

6. Результат ( ResultServlet ):
- показує результат проходження історії,
- дозволяє завершити сесію.

8. Профіль ( ProfileServlet ):
- показує логін користувача,
- кількість пройдених кінцівок,
- список отриманих результатів.

У проекті реалізована Ієрархія класів, що дозволяє удобно по ньому перебігтися та не запутатися. Для кожної сутності окремо написані такі класи:
- Wrapper ( дозволя удобно хранити дані на стороні сервера )
- Dto ( дозволяє читати дані з Json фалів)

Також на проект удобно добавляти нові історії. Потрібно відреагудувати Stories.json ( Добавити нову історію та її опис ),
Створити новий Json файл де будуть вибори до цієї історії ( за типом Map<String ( Назва фази ), Map<String (Вибір), String (До чого він призведе)>> )
Цей файл потібно обов'язково назвать як і сторію в Stories.json
